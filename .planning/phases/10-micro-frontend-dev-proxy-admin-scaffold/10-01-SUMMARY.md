# Phase 10 Plan 01: MFE Proxy Config + Admin Scaffold Summary

**One-liner:** Turborepo MFE proxy via microfrontends.json routing web/api/admin with port injection, plus complete apps/admin Vite CSR SPA scaffold with TanStack Router basePath /admin and adminClient auth.

## What Was Built

### Task 1: microfrontends.json and dev script updates

Created `apps/web/microfrontends.json` with three application entries:
- `web` (@kubeasy/web) on port 3000 — no routing rules (catch-all)
- `api` (@kubeasy/api) on port 3001 — routes `/api/:path*`
- `admin` (@kubeasy/admin) on port 3002 — routes `/admin` and `/admin/:path*`

Updated dev scripts:
- `apps/web`: `vite dev --port $(turbo get-mfe-port)`
- `apps/api`: `PORT=$(turbo get-mfe-port) tsx --env-file=.env --watch src/index.ts`

### Task 2: apps/admin scaffold

Created the complete `apps/admin` Vite CSR SPA package:
- **package.json**: `@kubeasy/admin`, Vite 8 + TanStack Router 1.168.3 + React 19 + better-auth 1.5.6
- **vite.config.ts**: `base: "/admin/"`, TanStackRouterVite plugin, @tailwindcss/vite, port 3002
- **tsconfig.json**: extends `@kubeasy/typescript-config/react.json`, vite/client types
- **src/main.tsx**: createRouter with `basePath: "/admin"`, QueryClientProvider
- **src/styles/globals.css**: Tailwind v4 + all @theme tokens from @kubeasy/ui + neo-brutalist utilities (no prose-neo, no web-specific animations)
- **src/lib/auth-client.ts**: adminClient() only, baseURL points to proxy localhost:3024
- **src/routes/__root.tsx** and **index.tsx**: minimal placeholders for TanStack Router
- **src/routeTree.gen.ts**: manually created minimal route tree (regenerated by Vite plugin on run)
