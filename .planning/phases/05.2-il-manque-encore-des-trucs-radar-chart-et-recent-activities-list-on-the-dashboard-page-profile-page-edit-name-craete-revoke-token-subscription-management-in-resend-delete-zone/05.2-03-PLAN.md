   **getEmailTopicsFn** — `createServerFn({ method: 'GET' }).handler(async () => { ... })`:
   - Get session by calling authClient.getSession with Cookie header forwarded (same pattern as auth.functions.ts)
   - Import `Resend` from 'resend', initialize with `process.env.RESEND_API_KEY`
   - Import `drizzle` from 'drizzle-orm/node-postgres' and `Pool` from 'pg' — create a database connection using `process.env.DATABASE_URL` to query the emailTopic table directly (server function runs on the web server, not the API server, so it needs its own DB connection)
   - **IMPORTANT**: Use a module-level lazy singleton for the DB pool to avoid creating a new connection per request:
     ```typescript
     let _pool: Pool | null = null
     function getPool() {
       if (!_pool) _pool = new Pool({ connectionString: process.env.DATABASE_URL })
       return _pool
     }
     ```
   - Query all rows from `email_topic` table using raw SQL: `SELECT id, resend_topic_id, name, description, default_opt_in FROM email_topic`
   - If no session (user not authenticated), return topics with `subscribed: defaultOptIn`
   - If session exists, query `resend_contact_id` from `user` table WHERE `id = session.user.id` using raw SQL
   - If no resendContactId or no RESEND_API_KEY, return topics with `subscribed: defaultOptIn`
   - Otherwise, call Resend contacts.topics.list({ id: resendContactId }) to get subscription status
   - Map topic subscriptions: for each emailTopic, look up subscription from Resend response. If found, use `subscription === "opt_in"`. Otherwise default to `defaultOptIn`.
   - Wrap Resend call in try/catch — on error, log and return default opt-in status (graceful degradation)
   - Return type: `Array<{ id: number; resendTopicId: string; name: string; description: string | null; defaultOptIn: boolean; subscribed: boolean }>`

   **updateEmailSubscriptionFn** — `createServerFn({ method: 'POST' }).validator((data: { topicId: number; subscribed: boolean }) => data).handler(async ({ data }) => { ... })`:
   - Get session (same pattern), throw if not authenticated
   - Query emailTopic by id to get resendTopicId
   - Query user's resendContactId
   - If no resendContactId, throw Error("No Resend contact found")
   - Initialize Resend, call `resend.contacts.topics.update({ id: resendContactId, topics: [{ id: topic.resendTopicId, subscription: data.subscribed ? "opt_in" : "opt_out" }] })`
   - Return `{ success: true }`

   - Import `useQuery`, `useMutation`, `useQueryClient` from `@tanstack/react-query`

   - Import Mail from lucide-react

   - Use `useQuery({ queryKey: ["email", "topics"], queryFn: () => getEmailTopicsFn() })` for topics data
   - Optimistic update mutation:
     ```typescript
     const updateMutation = useMutation({
       mutationFn: (vars: { topicId: number; subscribed: boolean }) =>
         updateEmailSubscriptionFn({ data: vars }),
       onMutate: async (vars) => {
         await queryClient.cancelQueries({ queryKey: ["email", "topics"] })
         const prev = queryClient.getQueryData(["email", "topics"])
         queryClient.setQueryData(["email", "topics"], (old: typeof topics) =>
           old?.map(t => t.id === vars.topicId ? { ...t, subscribed: vars.subscribed } : t)
         )
         return { prev }
       },
       onError: (_err, _vars, context) => {
         queryClient.setQueryData(["email", "topics"], context?.prev)
         toast.error("Failed to update email preferences")
       },
       onSettled: () => queryClient.invalidateQueries({ queryKey: ["email", "topics"] }),
       onSuccess: () => toast.success("Email preferences updated"),
     })
     ```
   - JSX: neo-brutalist card (`bg-secondary neo-border neo-shadow p-6`)
     - Header: Mail icon + "Email Preferences" title + subtitle "Control what emails you receive"

       - Left side: topic name (font-black) + description (text-sm text-muted-foreground)
       - Right side: Switch component, checked={topic.subscribed}, onCheckedChange calls updateMutation.mutate
       - Disabled switch when mutation is pending for that topic
