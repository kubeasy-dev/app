# Phase 05.2 Plan 01: Dashboard Radar Chart and Recent Activity Summary

**One-liner:** Recharts RadarChart with theme-name axes showing per-theme completion %, plus a 4-item activity preview with grouped-by-month dialog.

## What Was Built

- **DashboardChart** (`apps/web/src/components/dashboard-chart.tsx`): Renders a `ResponsiveContainer > RadarChart` with `PolarGrid`, `PolarAngleAxis` (theme names), `PolarRadiusAxis` (0-100 domain), and a `Radar` filled at `#6366F1`. Below the chart shows a "best theme" callout or empty-state message. Uses `useSuspenseQuery` for both `completionOptions({ splitByTheme: true })` and `themeListOptions()` to resolve theme slugs to human-readable names.

- **DashboardRecentActivity** (`apps/web/src/components/dashboard-recent-activity.tsx`): Shows first 4 XP transactions inline. Each `ActivityItem` has a left border accent, a challenge link, XP badge, and date. A "View All Activity" button opens a `Dialog` with a scrollable body (`max-h-[55vh]`) grouping all transactions by calendar month via `groupByMonth()`.

- **Dashboard route** updated to import and render both components, with `themeListOptions()` added to the loader `Promise.all` for SSR prefetching.
