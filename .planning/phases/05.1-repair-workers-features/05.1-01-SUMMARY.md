## Decisions Made

- USER_LIFECYCLE renamed to USER_SIGNIN — reflects that the queue fires on sign-in events (not generic lifecycle), aligns with UserSigninPayload semantics
- ChallengeSubmissionPayload drops xpAwarded/isFirstChallenge — workers now compute XP themselves, removing coupling between submission trigger and award logic
- XpAwardPayload gains description + challengeSlug — workers need these to write meaningful XP transaction records
- UserSigninPayload gains provider field — needed by user-signin worker to personalize welcome email / track sign-in source
- queryKeys live in api-schemas — shared between web app and any future consumer, avoids duplication
- Onboarding schema drops all Vercel Workflow webhook columns — Vercel Workflow replaced by BullMQ workers, no webhook-based step coordination
