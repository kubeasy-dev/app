## Decisions Made
- `CliEnv` type declared locally in `cli/index.ts` as `{ Variables: { user: SessionUser; session: null } }` to match `apiKeyMiddleware` injection shape. Using the broader `AppEnv` (where user is nullable) caused TypeScript errors requiring null-checks that are semantically wrong (apiKeyMiddleware guarantees non-null user). Decision follows the established pattern from STATE.md: "Sub-router env types: each Hono sub-router must declare its own Variables type".
